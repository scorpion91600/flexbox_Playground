🏗️ Structure de base

<!DOCTYPE html>

Déclare le type de document. Doit être la toute première ligne du fichier HTML.

Exemple : 
<!DOCTYPE html>
<html>

Élément racine de la page. Tout le contenu est à l'intérieur.

Attribut utile : lang="fr" → définit la langue de la page 
<html lang="fr">
  ...
</html>
<head>

Contient les métadonnées (non visibles sur la page) : titre, encodage, liens CSS...

<head>
  <meta charset="UTF-8">
  <title>Ma page</title>
</head>
<body>

Contient tout le contenu visible de la page.

<body>
  <h1>Bonjour !</h1>
</body>
<meta>

Balise auto-fermante pour les métadonnées. Plusieurs usages :

<!-- Encodage -->
<meta charset="UTF-8">

<!-- Responsive -->
<meta name="viewport" 
      content="width=device-width, 
               initial-scale=1.0">

<!-- Description SEO -->
<meta name="description" 
      content="Mon super site">
<title>

Définit le titre de l'onglet dans le navigateur. Important pour le SEO.

<title>Mon Site Web</title>
<link>

Lie un fichier externe à la page (CSS, favicon...).

<!-- Lier un CSS -->
<link rel="stylesheet" href="style.css">

<!-- Favicon -->
<link rel="icon" href="favicon.ico">
<script>

Intègre du JavaScript. Se place de préférence avant </body>.

<!-- JS externe -->
<script src="app.js"></script>

<!-- JS inline -->
<script>
  console.log("Hello !")
</script>

✍️ Texte & Titres

<h1> → <h6>

Les titres, du plus important (h1) au moins important (h6). Un seul h1 par page !

<h1>Titre principal</h1>
<h2>Sous-titre</h2>
<h3>Section</h3>
<h4>Sous-section</h4>
<h5>...</h5>
<h6>Plus petit titre</h6>
<p>

Définit un paragraphe. Crée un espace avant et après automatiquement.

<p>Ceci est un paragraphe.</p>
<strong>

Texte important (affiché en gras). A une valeur sémantique.

<p>C'est <strong>important</strong> !</p>
<em>

Texte avec emphase (affiché en italique). A une valeur sémantique.

<p>C'est <em>vraiment</em> cool.</p>
<span>

Conteneur inline générique. Utile pour styler une partie de texte avec CSS.

<p>Texte <span class="rouge">coloré</span></p>
<br>

Saut de ligne. Balise auto-fermante. À utiliser avec modération.

<p>Ligne 1<br>Ligne 2</p>
<hr>

Ligne horizontale de séparation. Balise auto-fermante.

<p>Section 1</p>
<hr>
<p>Section 2</p>
<blockquote>

Citation longue en bloc. Souvent indenté par défaut.

<blockquote cite="source.com">
  "La connaissance s'acquiert par l'expérience."
</blockquote>
<code>

Affiche du code informatique inline avec une police monospace.

<p>Utilise <code>console.log()</code></p>
<pre>

Texte préformaté : conserve les espaces et retours à la ligne.

<pre>
  function hello() {
    console.log("Hello !")
  }
</pre>

🔗 Liens

<a>

Crée un lien hypertexte. L'attribut href est obligatoire.

href → URL de destination

target="_blank" → ouvre dans un nouvel onglet

rel="noopener" → sécurité avec _blank

<!-- Lien externe -->
<a href="https://google.com" 
   target="_blank" 
   rel="noopener">Google</a>

<!-- Lien interne -->
<a href="page2.html">Page 2</a>

<!-- Ancre (même page) -->
<a href="#section1">Aller à section 1</a>

<!-- Email -->
<a href="mailto:test@mail.com">Email</a>

<!-- Téléphone -->
<a href="tel:+33600000000">Appeler</a>

📋 Listes

<ul>

Liste non ordonnée (puces). Contient des <li>.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
<ol>

Liste ordonnée (numérotée). Contient des <li>.

start="5" → commence à 5

reversed → compte à rebours

type="A" → A, B, C... (ou I, i, 1)

<ol>
  <li>Étape 1</li>
  <li>Étape 2</li>
  <li>Étape 3</li>
</ol>
<li>

Élément d'une liste (list item). Toujours enfant de ul ou ol.

<dl> <dt> <dd>

Liste de définitions : dl = liste, dt = terme, dd = définition.

<dl>
  <dt>HTML</dt>
  <dd>Langage de structure</dd>
  <dt>CSS</dt>
  <dd>Langage de style</dd>
</dl>

📊 Tableaux

<table>

Conteneur principal du tableau.

<thead> <tbody> <tfoot>

Sections du tableau : en-tête, corps, pied. Améliorent la lisibilité et l'accessibilité.

<tr> <th> <td>

tr = ligne | th = cellule d'en-tête (gras) | td = cellule normale

colspan="2" → fusionne 2 colonnes

rowspan="2" → fusionne 2 lignes

<table>
  <thead>
    <tr>
      <th>Nom</th>
      <th>Âge</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>25</td>
    </tr>
  </tbody>
</table>
<caption>

Titre/légende du tableau. Se place juste après <table>.

<table>
  <caption>Liste des élèves</caption>
  ...
</table>

📝 Formulaires

<form>

Conteneur du formulaire.

action="url" → où envoyer les données

method="GET/POST" → méthode d'envoi

<form action="/submit" method="POST">
  ...
</form>
<input>

Champ de saisie. Balise auto-fermante. Son comportement change avec type.

<!-- Texte -->
<input type="text" placeholder="Nom">

<!-- Email -->
<input type="email" required>

<!-- Mot de passe -->
<input type="password">

<!-- Nombre -->
<input type="number" min="0" max="100">

<!-- Date -->
<input type="date">

<!-- Checkbox -->
<input type="checkbox" checked>

<!-- Radio -->
<input type="radio" name="choix" value="oui">

<!-- Fichier -->
<input type="file">

<!-- Bouton submit -->
<input type="submit" value="Envoyer">

<!-- Champ caché -->
<input type="hidden" value="valeur">
<label>

Étiquette d'un champ. Le for doit correspondre à l'id de l'input.

<label for="prenom">Prénom :</label>
<input type="text" id="prenom" name="prenom">
<textarea>

Zone de texte multi-lignes.

<textarea rows="5" cols="30" 
  placeholder="Votre message...">
</textarea>
<select> <option>

Menu déroulant.

<select name="pays">
  <option value="">Choisir...</option>
  <option value="fr">France</option>
  <option value="be" selected>Belgique</option>
</select>
<button>

Bouton cliquable. Plus flexible que <input type="submit">.

type="submit" → soumet le form

type="reset" → réinitialise le form

type="button" → bouton simple (JS)

<button type="submit">✅ Envoyer</button>
<button type="reset">🔄 Réinitialiser</button>
<fieldset> <legend>

Groupe des champs liés avec un encadré et un titre de groupe.

<fieldset>
  <legend>Informations personnelles</legend>
  <input type="text" placeholder="Nom">
  <input type="email" placeholder="Email">
</fieldset>

🖼️ Médias

<img>

Affiche une image. Balise auto-fermante. alt est obligatoire pour l'accessibilité.

src → chemin de l'image

alt → description (accessibilité + SEO)

width / height → dimensions

loading="lazy" → chargement différé

<img src="photo.jpg" 
     alt="Description de la photo"
     width="300"
     loading="lazy">
<video>

Intègre une vidéo.

<video width="320" controls>
  <source src="video.mp4" type="video/mp4">
  Votre navigateur ne supporte pas la vidéo.
</video>

controls → affiche les contrôles

autoplay → démarre auto

muted → muet

loop → boucle

<audio>

Intègre un fichier audio.

<audio controls>
  <source src="musique.mp3" type="audio/mpeg">
</audio>
<iframe>

Intègre une page externe (YouTube, Google Maps...).

<iframe 
  src="https://www.youtube.com/embed/xxxxx"
  width="560" 
  height="315"
  allowfullscreen>
</iframe>
<figure> <figcaption>

Groupe une image avec sa légende. Sémantiquement correct.

<figure>
  <img src="photo.jpg" alt="Coucher de soleil">
  <figcaption>Coucher de soleil à Nice</figcaption>
</figure>

🧱 Balises Sémantiques

💡 Les balises sémantiques donnent du sens au contenu. Elles sont importantes pour le SEO et l'accessibilité.

<header>

En-tête de la page ou d'une section. Contient souvent le logo, la nav...

<nav>

Bloc de navigation principal (menu, liens de navigation).

<nav>
  <a href="/">Accueil</a>
  <a href="/about">À propos</a>
</nav>
<main>

Contenu principal de la page. Un seul par page.

<section>

Section thématique d'une page. Doit avoir un titre (h2, h3...).

<article>

Contenu autonome et réutilisable (article de blog, carte produit...).

<aside>

Contenu secondaire lié au contenu principal (sidebar, pub, liens...).

<footer>

Pied de page. Contient souvent les mentions légales, liens, contact...

<div>

Conteneur bloc générique, sans valeur sémantique. À utiliser quand aucune balise sémantique n'est appropriée.

<div class="container">
  <p>Contenu...</p>
</div>
<details> <summary>

Crée un accordéon natif sans JavaScript.

<details>
  <summary>Cliquez pour voir plus</summary>
  <p>Contenu caché révélé au clic !</p>
</details>
